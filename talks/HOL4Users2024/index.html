How Ethereum works

Ethereum is a shared decentralised computer.
There is global consensus on the state of the machine.
State updates obey deterministic rules, driven by user txns.

The state is organised into accounts, containing:
  • a 20 byte address
  • an ether balance
  • storage: a map from bytes32 to bytes32
  • (optionally) code: a sequence of bytes
Externally-owned account (EOA)
has private key associated with address, but no code
Smart contract account
has code, but no private key

How Ethereum works: message calls

An account can be called by another account.

Calls, used for both transfers and compute, contain:
  • sender address
  • target address
  • (optional) ether value to transfer
  • (optional) data
The target's code is executed with the call data as input
  • execution context for stack-based virtual machine with RAM
  • may spawn internal calls (new contexts), which return data
  • transactional: all or none of the storage and balance updates persist
A transaction (txn) from an EOA is a call
transaction info signed with the private key, including fee limit: fees (gas) are charged for execution and deducted from the sender account

How Ethereum works: blocks

state updates are an ordered sequence of transactions these transactions are organised into blocks this is for the sake of the consensus mechanism which ultimately designates certain blocks (at the head of a chain, since blocks refer to their parent) as final (i.e. canonical) peer-to-peer network of nodes have a view on the current machine state, but all agree on history at least till the finalised block transactions are broadcast. a (validating) node is randomly selected every slot to propose a block that specifies the included transactions and their order.

What is Verifereum?

formal verification for ethereum applications why this domain is good for verification: well-specified, small programs, very high value exploits that are essentially impossible to undo, immutable by default motivations: auditing market charges good money many applications have large bug bounties correct-by-construction application development may be worthwhile
